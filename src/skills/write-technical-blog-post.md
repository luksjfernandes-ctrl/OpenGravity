# Skill: Write Technical Blog Post

## Quando usar
Quando o usuário pedir para escrever um post técnico, artigo, ou análise aprofundada sobre implementação de software.

## Processo

### 1. Pesquisar o tópico
Use `browse_url` para reunir informações relevantes. Pesquise:
- Código-fonte e implementações relacionadas
- Documentação existente
- Exemplos e casos de uso
- Comentários que expliquem decisões de design

### 2. Identificar o ângulo interessante
Antes de escrever, responda:
- **Que problema isso resolve?** Não "o que faz", mas "o que daria errado sem isso?"
- **O que é surpreendente ou não-intuitivo?** A abordagem óbvia que não funciona, ou a complexidade escondida.
- **Qual é o insight-chave?** O "aha!" que faz a solução funcionar.
- **O que foi tentado primeiro?** Qualquer jornada ou iteração visível.

### 3. Escrever o rascunho (800-1500 palavras)
Estrutura:
1. **Enquadre o problema** — Hook com contexto e tensão
2. **Mostre o insight** — A ideia-chave que faz funcionar
3. **Caminhe pela implementação** — Código e explicação, aumentando complexidade
4. **Conclusão** — Onde vive no código, tradeoffs, links

### 4. Auto-avaliar
Checklist:
- [ ] Abertura enquadra um problema antes da solução?
- [ ] Há um momento "aha!" claro?
- [ ] É ancorado em implementação real, não abstrato?
- [ ] Exemplos de código constroem entendimento?
- [ ] Tom: pessoal e direto, sem enrolação?
- [ ] Links para código real?
- [ ] Profundidade adequada ao tema?

### 5. Apresentar
Envie o rascunho ao usuário para revisão. Itere conforme feedback.
